Color Defense

by: Jerry Huber

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I liked the idea of a game like Bloons Td 6, and games like it. It did not seem too complicated but still challenging in some of the components. And I wanted to make a game that I would probably play in the future with my own free time.

2. What I Learned

I got feedback like, "This tower is too good" or "You don't get enough money for the price" or "The game doesn't get challenging enough later into the game". All of these pieces of feedback led to me adjusting values in the game or adding more difficulty later into the game.

3. Architecture of My Game

The main parts of the game are: the towers, the enemies, the base, and the spawners. The towers are a stationary block that shoot projectiles, another class, that deals damage based on the towers statistics. The enemies are moving towards your base with the goal of destroying it and will die to projectiles or killing themselves on your base. The base is essentially the player's health bar and the spawners are where the enemies come from.

4. Technical Implementation

I made different classes for each tower that would be in the game. Each tower extended the main tower code that had all of the basic mechanics while the different tower classes would change values and maybe add some more to the tower. To upgrade to different towers, it destroys the tower that was previously in that location and makes a new tower that is what the player upgraded it to.

5. Most Challenging Part

One of the most challenging parts of the project was making sure that for one of the enemies, when it died, that it would spawn other enemies. That was very difficult because the path finding algorithm decided that it would break most of the time for those particular enemies. I overcame this by eventually releasing the reason why the enemies' path finding was breaking and changing the type of enemy to a slower one so that the values for when the enemy was supposed to change which tile it was going to would match the enemies' positions.

6. Future Fixes and Updates

I would spend more time on graphics and I would also spend more time on making the game more exciting or just harder.
